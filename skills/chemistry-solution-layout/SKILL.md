# Chemistry Solution Layout

將五個角色視為同一個 Skill 的五個內部階段，不要讓它們各自重寫輸出：

1. 化學內容階段：確認方程式、係數、電荷、單位、有效數字與推理中的客觀化學錯誤。只修正客觀錯誤，不改文字風格與解題順序。
2. 記號階段：統一 LaTeX、KaTeX、mhchem、分式、上下標、科學記號、單位與反應箭頭。保持原本數值、答案與推理。
3. NOTE 階段：在題目給定量、單位換算、重要變數、中間量、最後數值、化學式或係數上建立 NOTE。使用 `\htmlData{note=簡短說明}{標記內容}`；說明使用繁體中文、2～10 字、直接描述意義，不改公式。
4. 版面階段：只整理 DOM／CSS 結構與視覺間距。選項採固定二欄；所有一般行與分式行使用同一套行距；不得因分式或 NOTE 另行撐開整段。
5. 驗證階段：檢查公式、化學式、反應式、NOTE、選項、上下標、分式、溢位與手機版。驗證失敗時只回報真實問題；在實作專案時，才由程式修正後重新驗證。

## 必須遵守的版型

### 選項

- 依題目實際辨識到的數量輸出 `(A)`、`(B)`、`(C)`……，不要假設只有 A～E。
- 每個選項是一個二欄列：左欄只有標籤，右欄放完整分析。
- 選項換行、計算或理由延續時，全部從右欄正文開始，不得從標籤下方開始。
- 多選題逐項分析；每項寫出理由或計算，並在結尾明確判定正確／錯誤。
- 逐項判定必須與最終答案完全一致。

### 文字與行距

- 中文自然換行；不要用全段 `nowrap`。
- 短公式留在文字行內；過長公式使用單獨公式行並允許該公式容器水平捲動。
- 所有文字行、計算行、選項行共用固定行距；禁止建立「分式專用大間距」。
- 不得產生頁面級水平捲軸、巢狀水平捲軸、文字重疊或公式裁切。

### 數學與化學

- 分式使用 `\dfrac{分子}{分母}`。
- 化學式與反應式使用 `\ce{...}`，例如 `\ce{2H2 + O2 -> 2H2O}`、`\ce{Fe^{2+}}`。
- 科學記號使用 `\times 10^{-n}`；乘號不可寫成普通 `x` 或錯誤 LaTeX。
- 上下標必須使用大括號；公式中的單位應以 `\mathrm{}` 或 `\,` 正確呈現。
- 所有 `$` 必須成對；不得讓裸 LaTeX、裸化學式或反應箭頭流入畫面。

### NOTE

- NOTE 的底色是主要識別方式；不要依賴虛線或陰影。
- NOTE 內容只回答「這個數字／符號代表什麼」，不要塞入整段教學。
- 相同數值若在不同步驟代表不同意義，建立不同 NOTE。
- NOTE 不得包住破碎的分式、單獨指數或會破壞 KaTeX 結構的半截 token；平方項需整體包住，例如 `\htmlData{note=濃度平方}{(2x)^2}`。

## 角色邊界

- 內容階段不可改排版、NOTE、HTML、CSS、JavaScript。
- 記號階段不可改化學正確性、推理、答案或排版。
- NOTE 階段只能新增或修正 NOTE metadata，不重寫公式與說明。
- 版面階段不可改化學、數學、答案或 NOTE 語意；必要時可調整既有 DOM 包裝以保證不重疊。
- 驗證階段不可偷偷重寫詳解；只回報或觸發明確的修正流程。

## 執行方式

收到化學詳解或專案排版問題時：

1. 先辨識輸出是純文字詳解還是 `@@BOARD@@ ... @@END@@` BoardDoc。
2. 保留原內容與答案，按五階段順序處理。
3. 先處理選項群組，再處理公式渲染與 NOTE，最後套用固定版面規則。
4. 用桌面與手機寬度檢查選項欄位、分式、NOTE 與長反應式。
5. 發現問題只修正對應階段，避免為了排版改動化學或答案。

## 交付標準

只有在以下條件都成立時才視為完成：

- 選項標籤與正文欄位對齊，換行不回到標籤下方。
- 普通行與分式行距一致，且沒有重疊或遮蔽。
- NOTE 有底色、可互動、說明簡短，且沒有不必要虛線。
- 化學式、上下標、分式、反應箭頭正常渲染。
- 桌面與手機沒有頁面級水平溢位。
- 最終答案、化學內容與原推理意義未被排版流程改變。
