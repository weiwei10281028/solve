# 化學通則卡格式（CARD_FORMAT）

寫卡 AI：保留下列**必要標題與順序**；各標題下條列可自由加長。  
需要多寫時：加條列，或加在文末「擴充」區（可多個 `# 擴充_...`）。  
不要寫成論文或大段落。

解題 AI：優先讀 **關鍵通則 → 判定條件 → 解題步驟**，其餘當補充。

---

## 檔名

- `{id}.en.md`（解題主檔；條列可用英文短句，標題用下列中文）
- `{id}.zh.md`（可選；給人對照）

---

## 完整範本

```markdown
---
format_version: "3.2"
id: your-id
card_version: "1.0.0"
lang: en
title_en: "Short English name"
title_zh: "短中文名"
priority: 50
status: draft
triggers:
  any:
    - "keyword1"
    - "秒錶反應"
    - "SpeciesA"
  all: []
  exclude:
    - "易混淆但不屬本類型"
---
```

---

## 各區寫什麼（無條數上限）

| 區 | 寫什麼 |
|----|--------|
| 關鍵通則 | 解題最先套用的判定句 |
| 試劑角色 | 氧化劑、還原劑、指示劑等 |
| 觀察現象 | 巨觀變化；會／不會出現的條件 |
| 化學計量 | 莫耳數比、邊界、限量試劑 |
| 實驗條件 | 溫度、酸鹼性、濃度、混合等 |
| 反應步驟 | 反應先後與快慢（事實） |
| 解題步驟 | 有順序的檢查／計算 |
| 判定條件 | 若不成立 → 則（不可外推時間等） |
| 解題禁忌 | 判定通過前不可做 |
| 常見錯誤 | 錯 → 正 |
| 常見變體 | 不同試劑組合的差異 |
| 擴充 | 日後加內容，不拆主結構 |

YAML 的 `triggers`＝**辨識關鍵詞**（系統欄位名可維持英文）。

---

## 文風

- 壓縮句：`莫耳數比 > 1:3 → 變藍`  
- 數字、不等式優先  
- 需要完整就多寫；不要為短而漏掉會改答案的條件  
- 同一事實不要改成長段重複  

---

## 擴充方式

1. 保持同一 `id`  
2. 提高 `card_version`（加內容用 `1.1.0`；改判定條件語意用 `2.0.0`）  
3. 在原標題下追加條列，或新增 `# 擴充_...`  
4. 更新 `_INDEX.md` 的版本號
