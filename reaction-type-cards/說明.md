# 化學通則卡 — 說明

## 正式名稱

**化學通則卡**

- 英文檔名／系統用名：Reaction Type Card  
- 口語可簡稱「通則卡」；文件請用 **化學通則卡**

含義：某一類化學題（反應、實驗、緩衝溶液、電化學…）的**可判定通則**，給解題 AI 先套用；不是單題答案，也不是講義。

## 用語（台灣常用）

| 本系統用語 | 意思 |
|------------|------|
| 關鍵通則 | 解題時最先套用的判定句 |
| 試劑角色 | 氧化劑、還原劑、指示劑等 |
| 觀察現象 | 題目要看的顏色、沉澱等變化 |
| 化學計量 | 莫耳數、莫耳數比、限量試劑 |
| 實驗條件 | 溫度、酸鹼性、濃度、混合方式等 |
| 反應步驟 | 反應先後／快慢（條列事實） |
| 解題步驟 | 依序該算／該查什麼 |
| 判定條件 | 若不成立，不可外推時間或斷言會出現該現象 |
| 解題禁忌 | 判定尚未通過前不可做的事 |
| 常見錯誤 | 典型跳步驟與正確作法 |
| 常見變體 | 同類實驗的不同試劑組合 |
| 辨識關鍵詞 | 用來判斷該掛哪一張通則卡 |
| 擴充 | 之後加內容的區塊 |

（不用「閘門」等非課堂用語。）

## 目的

用精闢條列寫出：試劑角色、觀察現象成立與否、化學計量邊界、實驗條件、判定條件與解題禁忌，讓 AI **先判定現象成不成立，再算反應速率或時間**。

## 檔案

| 檔案 | 用途 |
|------|------|
| `說明.md` | 本說明 |
| `CARD_FORMAT.md` | 格式與基本規則 |
| `WRITE_CARD_PROMPT.md` | 寫卡提示詞（給其他 AI） |
| `_INDEX.md` | 化學通則卡目錄 |
| `{id}.en.md` | 解題用主檔 |

## 流程

### 產卡

1. 打開 `WRITE_CARD_PROMPT.md`，複製提示詞  
2. 附上 `CARD_FORMAT.md`  
3. 填類型名稱（可加幾條已知通則）  
4. 其他 AI 產出 `{id}.en.md` → 放入本資料夾 → 更新 `_INDEX.md`  
5. 需要評析再交給本專案 AI

### 解題

1. 依 `_INDEX`／辨識關鍵詞選一張化學通則卡  
2. 只提供 `{id}.en.md`  
3. 先讀「關鍵通則」「判定條件」，再算數

```text
請套用對應的化學通則卡（.en.md）。
先依「關鍵通則」「判定條件」判斷觀察現象是否成立；
條件不成立時：不可給出變色／完成時間，也不可只憑濃度比外推時間。
通過後才可討論反應速率與時間。
```

寫卡格式與提示詞：`reaction-type-cards/`（產卡工具）。  
**解題實際注入的通則卡庫**：`chem-rule-cards/`（與章節類型分開；**依題意預判**參考，非關鍵字主路徑）。

新增通則卡後：更新 `chem-rule-cards/` 與執行 `python scripts/build-chem-rule-cards-js.py`。
