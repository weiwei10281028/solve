# 寫卡提示詞（給其他 AI）

複製下方提示詞給寫卡 AI。  
同時附上：`CARD_FORMAT.md`。  
不要附單題詳解；只給類型名稱與你已知的通則（可空）。

本系統正式名稱：**化學通則卡**。標題與說明請用台灣高中化學常用語（氧化劑、還原劑、指示劑、莫耳數比、限量試劑、觀察現象、實驗條件、判定條件…）；**不要用「閘門」**。

---

## 提示詞（複製區）

```text
你要寫「一張化學通則卡」，給解題 AI 使用（機器可套用的通則，不是給人讀的教科書）。

請嚴格依照 CARD_FORMAT.md：保留必要中文標題與順序。
各標題下條列可盡量寫完整；若還要加料，只能加在「擴充」或「擴充_…」區塊，方便之後繼續擴充。

任務——寫到解題夠用為止：
- 一律精闢條列，一行一個事實。優先寫比值、不等式、會／不會成立的條件。
- 盡量涵蓋本類型解題需要的內容（有就寫）：
  - 氧化劑、還原劑、指示劑、中間產物、催化劑
  - 觀察現象是什麼；什麼條件會出現／不會出現
  - 關鍵化學計量與邊界（恰等於某比值、不足、過量）
  - 實驗條件：溫度上下限、酸鹼性、濃度範圍、混合順序、光照、催化劑、常見操作限制
  - 反應步驟（事實，不要說故事）
  - 有順序的解題步驟
  - 判定條件（寫成：若不成立：… → 則：…）
  - 解題禁忌（尤其：現象尚未確認前就外推速率或時間）
  - 常見錯誤（錯 → 正）
  - 重要變體（不同試劑／反應網路）及其邊界差異
- 不要早停：凡會改變「現成不成立／能不能給時間」的細節都要寫。
- 不要灌水：每一條都要能在解題時直接用。

基本限制：
- 條列，不要長篇段落。
- 不要寫單題答案、選項字母、年分或試卷代號。
- 內容必須對「同一類型的其他題」仍然成立。
- triggers.all 預設 []；除非那些字真的常出現在題幹。
- triggers.any 請同時放台灣高中常見中文詞與英文／化學式。
- 標題用 CARD_FORMAT 的中文；條列可用簡短英文或中英並陳（解題主檔 .en.md）。
- 解題順序：關鍵通則 → 判定條件 → 解題步驟，之後才算反應速率或時間；判定不成立 → 不可給秒數時間。

產出：
- 一個 {id}.en.md（若使用者要求再加 {id}.zh.md）。
- 填滿必要區塊；真的不適用才寫 n/a。
- 之後若要擴充：保持同一 id，提高 card_version，追加條列或「擴充」區塊，不要改寫成長文。

使用者主題：
- 類型名稱: <FILL>
- 已知通則（可空）: <FILL or none>

若關鍵事實不明，先問少數澄清問題，再寫完整通則卡。
```

## 使用者怎麼填

- `類型名稱`：中／英皆可  
- `已知通則`：你已知的精闢通則（可空）
