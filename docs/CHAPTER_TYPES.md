# 章節類型與作答格式

主頁的「章節類型」與「作答格式」使用同一份 `js/solve-spec.js` 規格，讓提示詞和回覆檢查器只依一個來源行事。

## 使用規則

- 章節可複選，最多三項；未選擇時系統僅顯示題幹偵測到的候選，不會強迫任何模板。
- **化學通則卡**（`chem-rule-cards/` + `js/chem-rule-cards.js`）與本章節選擇**完全分開**：解題前依**題意預判**注入【參考】通則，不經進階章節勾選。
- 已選章節仍會與題幹比對。題幹訊號明顯屬於其他章節時，該章節會標示為不適用並略過提示詞與檢查，避免例如酸鹼題被硬要求計量推導；若沒有足夠文字訊號（例如圖片題），保留為待判定並尊重使用者選擇。
- 作答格式開關已取消；所有題目固定使用【題意→依據與推導→結果】骨架，選擇題再逐項分析。
- 使用者選項只增加其明確要求，不能覆寫題目數據、核心正確性、既有輸出安全規則或把不適用章節硬套到答案。

## 目前章節目錄

| ID | 章節 |
|---|---|
| `atomic` | 原子結構與週期性 |
| `bonding` | 化學鍵與分子構型 |
| `stoichiometry` | 化學計量與反應式 |
| `gases` | 氣體 |
| `solutions` | 溶液與濃度 |
| `acidbase` | 酸鹼 |
| `redox` | 氧化還原與電化學 |
| `equilibrium` | 化學平衡 |
| `kinetics` | 反應速率 |
| `organic` | 有機化學 |
| `experiment` | 實驗、鑑定與安全 |

## 擴充方式

在 `CHAPTERS` 新增一項時，必須同時提供穩定 ID、顯示名稱、簡短說明、題幹關鍵字規則與解題提醒。介面與 `buildUserBlock()` 會使用同一筆資料，因此不應另行在頁面或提示詞檔硬寫章節名稱。
