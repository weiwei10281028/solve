# 提示詞優先順序與不可覆蓋規則

更新日期：2026-07-13（Phase 3）  
目的：界定提示詞的職責與組裝順序；本階段不更動既有提示詞內容或輸出格式。

## 模組邊界

| 模組 | 主要位置 | 職責 | 不應承擔的事 |
| --- | --- | --- | --- |
| 主提示詞 | `prompts/base/system-chem.md` | 教學語言、LaTeX、輸出格式、解題流程與共同風格。 | 不寫特定題目的數值、答案或知識庫命中內容。 |
| 網頁選項 | `js/app.js`、`js/solve-spec.js`、`js/prompts.js`、`prompts/addons/` | 章節類型、作答格式、詳細模式、計量表強制、計算精簡、解題範圍等使用者選項。 | 不可解除主提示詞的輸出格式、正確性與安全規則。 |
| 題型規則 | `js/prompts.js` 的意圖／範圍／選項判斷函式 | 根據題幹決定選擇、計算、問答、子題及圖片題的作答範圍。 | 不可憑規則推測缺失題幹、選項或數據。 |
| 教師知識與題庫 | `js/knowledge-store.js`、`js/knowledge-tools.js`、資料庫載入函式 | 載入與本題相關的教學規定、方法、格式與資料庫內容。 | 不可把不相干內容或過期答案當成題目事實。 |
| 參考答案 | `buildReferenceAnswerUserBlock`、`buildRefAnswerSystemAddon` | 初次解題的答案收斂目標，以及生成後的本機比對。 | 不可取代依題目重新推導；明顯衝突時須跳過錨定；不一致時只標示供複核，不可自動重寫詳解。 |
| 局部修正 | `buildQualityFixUserText`、`buildRefAnswerFixUserText` 與各 `*FixUserText` | 針對已辨識問題重寫完整答案。 | 不可擴大為另一套主提示詞、不可改變題目範圍或未標示的正確推導。 |

## 實際組裝順序

### 初次解題的 System 訊息

`getSystemPromptForSolve()` 以以下順序串接：

```text
主提示詞
  -> 詳細模式附加
  -> 追問模式附加（僅追問）
  -> 題目／子題／選項範圍
  -> 參考答案收斂目標（通過衝突檢查時）
  -> 化學反應方程式表達（選項開啟時）
  -> 計算精簡（選項開啟時）
  -> 教師知識庫的 system 規定
```

### 初次解題的 User 訊息

`buildSolveUserText()` 先組合範圍鎖定、題目（文字或圖片補充）、詳細模式與題型指示，再加入參考答案。`js/app.js` 隨後追加：

```text
教師知識庫 user 規定
  -> 題庫命中內容
  -> Board 格式提示
  -> NOTE 附錄
```

資料庫與 NOTE 是「依本題補充」：只應補上題幹未表達的教學規定或呈現要求，不能改寫題目條件、指定範圍或參考答案的衝突處理。

### 追問與品質修正

- 追問使用獨立的 `getSystemPromptForFollowUp()`／`buildFollowUpUserText()`，只回答新的追問，不重套整份解題模板。
- 單次品質修正保留原 System 訊息與原答案，最後加入修正指令；修正指令只能處理被檢查器列出的格式、NOTE 或答案一致性問題。
- 本機可安全處理的 LaTeX 修復留在渲染層，不應新增成模型提示詞。

## 不可覆蓋規則

1. 題目、圖片與使用者明確指定的題號／子題／選項決定解題範圍；附加規則不能擴大範圍或猜測缺漏。
2. 主提示詞的共同輸出契約（繁體中文、高中程度、完整 LaTeX、傳統詳解、`@@ANSWER@@`）不可由網頁選項、資料庫或局部修正取消。
3. 網頁選項只可增加其已命名的行為。兩項選項並存時，計量表先決定是否必須呈現表格，計算精簡只壓縮表格後的數值推導，不可刪除必要表格；`solveSpec` 只對使用者選取且題幹判定適用的章節／格式要求明確步驟，未選取時不得注入規格。
4. 題庫／教師規定只在命中本題時加入；若與主提示詞衝突，保留主提示詞共同契約，並將規定縮小為本題教學補充。
5. 參考答案是收斂目標而非題目事實。`checkObviousRefAnswerConflict()` 判定明顯衝突時，必須改依題目推導，且不可再注入參考答案錨定；生成後若比對不一致，只顯示本機複核結果，不能依參考答案自動改寫。
6. 局部修正必須保留未被點名的邏輯、數值、範圍與中文說明；不得藉修正引入新題型模板或資料庫規則。
7. 同一規則只能有一個權威來源。需要改主規則時改 `prompts/base/`；需要改選項規則時改對應 `prompts/addons/`；不要在 `prompts.js` 複製長篇規則。

## 維護流程

1. 先判斷規則屬於上表哪一個模組；跨模組規則須拆成各自最小的責任。
2. 修改 `prompts/` 後執行 `python scripts/rebuild-prompt-fallback.py`，使離線 fallback 與來源一致。
3. 若新增網頁選項，同時定義它的 System 附加層、User 簡短提示、與既有選項並存時的不可覆蓋關係。
4. 若新增品質檢查，修正訊息必須採「只修正列出問題」並以完整替代答案回覆。
5. 執行 `python tests/check-js-syntax.py` 與 `python tests/run-self-test.py`；對實際題目確認輸出沒有重複或衝突指令。
